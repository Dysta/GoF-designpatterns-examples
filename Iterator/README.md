# Iterator
**Type** : Patern Comportemental \
[Plus de détail](https://refactoring.guru/design-patterns/iterator)
## Intention
**Iterator** est un modèle de conception comportementale qui vous permet de parcourir les éléments d'une collection sans exposer sa représentation sous-jacente (liste, pile, arborescence, etc.).
